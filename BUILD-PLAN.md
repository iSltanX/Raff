# BUILD-PLAN — رفّ (Raff)

## Understanding

**Product** (from `clipboard-manager-plan-ar.md`, binding): a personal, single-user,
**local-only**, keyboard-first clipboard manager for macOS. Two layers only:

1. **الأخير** — an automatic capped history of everything copied.
2. **مثبّت** — a pinned shelf of frequently reused items (prompts, code, links, text); never auto-pruned.

One global hotkey (default **⇧⌘V**) toggles a Spotlight-style floating panel:
type to filter, **⏎** pastes into the previous app, **⌥P** pins, **esc** closes.
No account, no cloud, **zero network calls**. "Learning" in v1 is **silent signal
logging only** (`copyCount` / `pasteCount` / `lastUsedAt`) — no adaptive ranking;
sorting is simple recency. Settings expose the learning toggle, "عرض ما تعلّمه رفّ",
and "مسح بيانات التعلّم".

**Visual spec** (from `R/` — pixel-level): warm paper/wood palette with honey accent
(light `#D8913C` / dark `#E8B267`), frosted-glass panel (radius 14, blur 40 sat 180%),
44px rows, 8px grid, full RTL, Cairo (display) + Almarai (body), light + dark.
The React components in `R/_ref/identity` define the panel, tray dropdown, settings,
and first-run screens exactly; `R/_ref/icon` defines the app icon / menu-bar
template / symbol mark as SVG — these are rendered to real PNG/ICNS assets.

**The plan's §6 (Swift architecture) is disregarded**; same behavior is implemented
in Tauri v2 (Rust + vanilla HTML/CSS/JS).

## Architecture

- `src-tauri/` — Rust: `main.rs`, `monitor.rs` (350ms pasteboard poll thread),
  `paste.rs` (focus restore + ⌘V synthesis), `storage.rs` (JSON store),
  `commands.rs` (least-privilege IPC), `panel.rs` (NSPanel + vibrancy), `tray.rs`,
  `macos.rs` (thin AppKit boundary: changeCount, concealed flags, frontmost app,
  pasteboard read/write).
- `src/` — vanilla frontend: `index.html` (panel), `settings.html`, `firstrun.html`,
  `styles.css` (design tokens), JS modules (`logic.js` pure/testable, `store.js` IPC,
  `panel.js`, `settings.js`, `firstrun.js`), bundled `fonts/`.
- Plugins/crates: `tauri-plugin-global-shortcut`, `tauri-plugin-autostart`,
  `tauri-nspanel` (non-activating panel), `window-vibrancy` (HudWindow),
  `core-graphics` (⌘V CGEvent), `objc2-app-kit` (pasteboard/workspace boundary).
- Storage in app-data dir: `history.json` (capped), `pinned.json`, `settings.json`,
  `images/` (PNG + thumb per image item). Unreadable JSON → copied to `<name>.corrupt`
  before overwrite.
- Security: real CSP, capabilities limited to core defaults, all clip content rendered
  with `textContent`, activation policy Accessory (no Dock icon), zero network.
- Concealed pasteboard flags (`org.nspasteboard.ConcealedType`,
  `org.nspasteboard.AutoGeneratedType`) are never stored; per-app exclusion by bundle id.

## Phase order (commit after each; app runnable at every step)

- **Stage 0 — skeleton:** project scaffold, tokens + fonts, tray icon + menu,
  global hotkey, text capture (dedupe), floating NSPanel with list, paste into
  previous app, icons/identity assets. → daily-usable core.
- **Stage 1:** instant search/filter, paste as plain text (⌥⏎), delete item (⌘⌫),
  clear history, concealed-flag handling, corrupt-file guard.
- **Stage 2:** pinning (⌥P + pin toggle in row), pinned section, pinned persistence.
- **Stage 3:** images (PNG + thumbnails), per-app exclusion list, full Settings window
  (عام/الخصوصية/التعلّم/حول), first-run Accessibility screen, launch at login.
- **Stage 4 — finish:** Node + Rust tests, README.md, NOTES.md (feature ↔ plan-phase map),
  final QA pass (RTL, AA contrast, light/dark).
